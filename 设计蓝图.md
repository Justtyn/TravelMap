一、总体创意：城市智慧旅游 + 旅行商城一体化

项目名示例：**《CityGo 智慧旅游助手》**

- **APP1（课设一）**：城市旅游导览 APP
   功能：景点浏览、路线规划、高德地图导航、语音讲解、图文攻略、收藏等。
- **APP2（课设二）**：旅游商品 / 门票商城 APP
   功能：远程数据库读写、景点门票 / 旅行产品展示、购物车、下单、订单管理等。

两者可以：

- 共用同一个后台数据库（user、scenic、product等）。
- APP1 里“立即购票”按钮跳转到 APP2（理论设计里写“未来可整合为同一 APP 的不同模块”）。

同时满足：

- 高德地图支持 ✅
- 微信登录支持 ✅
- 远程数据库 CRUD ✅
- 图文并茂 + BGM + 动画特效 ✅
- 创意与特色 ✅（一整套智慧旅游方案，而不是单纯“展示信息”）

## 二、课设一：旅游项目 APP 设计（≥ 6 个页面）

### 1. 功能定位

**目标用户**：来某个城市旅游的大学生 / 年轻人。
 **核心卖点**：

- 一键查看城市景点 + 高德地图导航；
- 每个景点有**图文攻略 + 音频讲解 + 背景音乐**；
- 可以按主题路线（夜游、摄影、亲子）一键规划。

### 2. 页面规划（至少 6 页）

可以采用 BottomNavigation + Fragment 结构：

1. **启动 / 引导页（Splash + 引导）**
   - 内容：项目 Logo、渐变背景、简单动画（属性动画或 Lottie）。
   - 逻辑：
     - 首次安装显示 2–3 页引导轮播（ViewPager2），说明主要功能。
     - 之后直接进入首页。
   - 技术点：
     - 属性动画 / Lottie 动画（加“特效”分）。
2. **首页（推荐页）**
   - 内容：
     - 顶部轮播图 Banner（热门景点大图）。
     - 下方用 Card + RecyclerView 展示“热门路线”，每个卡片包含大图 + 标题 + 简介 + “去看看”按钮。
   - 交互：
     - 点击景点进入“景点详情页”。
     - 点击主题路线进入“路线详情页”。
   - 评分点：
     - 图文并茂、界面美观（良好/优秀加分）。
3. **景点列表页**
   - 内容：
     - 搜索框（按关键字查景点）。
     - 分类 Tab（自然、人文、博物馆、夜景…）。
     - RecyclerView 列表：景点缩略图 + 名称 +评分 + 简短介绍 + “地图”小图标。
   - 交互：
     - 下拉刷新（SwipeRefreshLayout）。
     - 点击列表项进入景点详情页。
4. **景点详情页（含语音、音乐、进度条）** ★★★（本课设加分核心）
   - 内容：
     - 顶部大图（ImageView）。
     - 简介、开放时间、票价信息。
     - “在地图中查看”按钮（跳转到地图页）。
     - **音频讲解播放器**：
       - 播放/暂停按钮
       - SeekBar 作为**进度条**显示播放进度
       - 当前时间 / 总时长
     - 背景音乐：
       - 可以在此页或整站公共播放轻音乐（用 MediaPlayer 循环播放）。
   - 技术点：
     - MediaPlayer + SeekBar 实现音频播放与进度更新（满足“有背景音乐，有进度条”硬性要求）。
     - 可以把“语音讲解”音频文件放在 raw 目录下。
5. **地图导航页（高德地图）**
   - 内容：
     - 高德 MapView / TextureMapView。
     - 显示当前位置 + 目标景点 Marker。
     - 底部展示“步行/驾车导航”按钮，点击启动高德 App 或内部导航。
   - 设计：
     - 进入此页时从 Intent 中获取景点经纬度。
     - 显示附近其它景点的小图标（多个 Marker），Marker 可以有放大/跳动动画（特效分）。
6. **路线规划 / 行程页**
   - 功能：
     - 用户可选择 3–5 个景点加入当天行程。
     - 显示为时间线或卡片列表（9:00 景点 A → 11:00 景点 B → …）。
     - 按“开始导航”时在地图上连线（Polyline），展示线路。
   - 创意：
     - 支持“一键生成拍照路线”、“夜景美食路线”等预设模板。
7. **个人中心 / 登录页（微信登录）**
   - 内容：
     - 用户头像、昵称（通过微信登录获取）。
     - 我的收藏路线、我的收藏景点。
     - 跳转到“旅游商城 APP”的入口按钮。
   - 技术思路（说明书写思路即可，代码可以简化）：
     - 使用微信开放平台 Android SDK。
     - 登录流程：APP 调用微信 → 微信授权 → 回调 code → 后端换取 openid/unionid → 生成本地 Token。
     - 数据库为 user 表保存 openid、昵称、头像 URL 等。

> ✅ 页面数量：Splash + 引导 + 首页 + 景点列表 + 景点详情 + 地图页 + 行程页 + 个人中心 = 完全够 6+ 页。

### 3. 课设一的特效 & 多媒体设计

- 进入首页时的卡片浮现动画（RecyclerView item 动画）。
- 点击卡片的共享元素转场（Activity/Fragment 过渡动画）。
- Lottie 动画用在“空空如也”、“加载中”状态。
- 背景音乐 + 音频讲解 + 进度条（主要加分点）。

## 三、课设二：访问远程数据库的“旅游商城 APP”

可以起名 **《CityGo 旅行商城》**，只做商城部分，和上面的导览APP逻辑相关联。

### 1. 功能定位

- 通过**远程数据库**存储和读取数据：景点门票、旅游路线产品、用户订单等。
- 实现：
  - 商品查询浏览
  - 商品详情
  - 加入购物车
  - 下单与订单列表
  - 后台 CRUD（添加 / 修改 / 删除商品）

### 2. 数据库与表设计（示例）

假设后端 MySQL + Spring Boot，仅要在说明书中画出 ER 图：

- `user`
  - id, openid, nickname, avatar, phone, create_time …
- `product`（旅游产品/门票）
  - id, name, cover_img, price, stock, type, desc, scenic_id, create_time, update_time
- `cart_item`
  - id, user_id, product_id, quantity, create_time
- `order`
  - id, user_id, total_price, status, create_time, pay_time
- `order_item`
  - id, order_id, product_id, quantity, price

> 在说明书中重点说明：**Android 通过 HTTP 请求访问后端 REST 接口**，实现远程数据库读写。

### 3. APP 主要页面与功能

1. **登录 / 启动页**
   - 与导览 APP 相同的微信登录思路（可简化为普通账号密码登录，说明书里写“支持微信登录扩展”）。
   - 登录成功后保存 Token，用于后续接口访问。
2. **商品列表页**
   - 从接口 `GET /api/products` 获取数据。
   - 显示商品图、名称、价格、类型、购买按钮。
   - 下拉刷新、分页加载。
3. **商品详情页**
   - 接口：`GET /api/products/{id}`。
   - 展示大图、详细介绍、价格、库存、关联景点信息。
   - 按钮：
     - “加入购物车”
     - “立即购买”
   - 点击加入购物车：调用 `POST /api/cart/add`（向数据库写入一条 cart_item 记录）——**满足“添加功能”**。
   - 还可以在详情页内播放景区宣传短音频/视频，继续保持“图文并茂+多媒体”。
4. **购物车页**（加分点）
   - 接口：`GET /api/cart/list`、`POST /api/cart/update`、`POST /api/cart/delete`。
   - 列表显示购物车中的商品，可修改数量或删除。
   - 下方显示总价，点击“去结算”。
5. **订单确认页 & 支付流程设计**（可以只做 UI + 流程说明）
   - 填写联系人信息（姓名、电话）。
   - 显示待支付金额。
   - 点击“微信支付”：
     - 实际上在课程实验里，可以不接真支付，仅模拟调用。
     - 说明书中写“调用微信支付 SDK 的预留接口，当前使用 Mock 支付成功”，同时更新数据库 `order.status` 为“已支付”。
6. **我的订单页**
   - 列表展示订单号、下单时间、金额、状态。
   - `GET /api/orders/my` 读取远程数据库订单记录。
7. **后台管理入口（选做，加分 +10）**
   - 简单做个“管理员登录”界面（也可以单独做 PC 端）。
   - 功能：添加新商品（调用 `POST /api/admin/products`）、修改商品信息、上下架等。
   - 只要实现一两个简单界面 + 调接口，就能体现“添加、修改”能力。

### 4. 架构与技术说明（远程数据库访问）

**前端（Android）**：

- 开发语言：Java 或 Kotlin（看老师课程要求，多数学校是 Java）。
- 网络访问：Retrofit + OkHttp + Gson。
- 图片加载：Glide / Coil。
- 本地缓存：Room / SharedPreferences（缓存登录信息、收藏记录等）。

**后端（可在说明书中简略说明）**：

- Spring Boot + MySQL（或老师课堂给的远程数据库接口）。
- 提供 REST API：
  - GET /products
  - GET /products/{id}
  - POST /cart/add
  - GET /cart/list
  - POST /orders/create
  - GET /orders/my
  - ……

在课程设计说明书里重点写清楚：

> “本 APP 通过 HTTP + JSON 的方式调用后端接口，后端负责对 MySQL 数据库进行增删改查，以实现旅游商品的线上展示与订单管理。”

7. **使用说明**
   - 用户如何安装 APK。
   - 常见操作步骤演示（用截图标注）。

   - 本次课设的收获与不足。

## 五、答辩时怎么讲，才能拿高分

答辩时可以按这个逻辑说：

1. **先讲创意和整体目标**

   > “我做的是一整套城市智慧旅游解决方案，包括旅游导览 APP 和旅行商城 APP。前者解决‘玩什么、怎么去’的问题，后者解决‘怎么买票’的问题。”

2. **再讲技术点与难点**

   - 高德地图集成：定位、Marker、路线规划。
   - 微信登录（可以重点说流程和安全性，即使代码不完全）。
   - 音频播放 + 进度条 + 背景音乐。
   - 远程数据库 + Retrofit 调用 + 购物车/订单管理。

3. **最后展示界面与特效**

   - 展示图文、动画、音频、地图、购物车流程。
   - 强调：“在界面美观、交互体验和功能完整性上做了较多优化，希望可以达到优秀标准。”
